01 Â· The Basics
What is it?
Claude Code is an agent that ships code with you â not a chatbot that answers about it.
It lives in your terminal, IDE, or browser â reads your project, plans the work, takes the action, and asks before it does anything risky.
it's an
Agentic loop.
Claude reads context, picks tools, edits files, runs commands, and verifies results â then loops again until the task is done.
it's also a
Workflow shift.
You move from typing every line to directing intent. The job becomes: be specific, set the constraints, review the diff.

Foundations Â· Step 2

Write a prompt that actually gets work back.

The first prompt sets the tone for the whole session. Spend 30 seconds being specific â you'll save 30 minutes of redirection later.

01
Be specific about the outcome.
"Add a hero image to blog.html using Brand Assets/Bloghero2.png as a left-text overlay" beats "make the blog page nicer."
02
Use Plan Mode for anything new.
Plan Mode shows you what Claude intends to do before touching a single file. Approve the plan, then let it run.
03
Pick the right approval mode.
Approval mode for new code paths. Auto-accept once you trust the loop. Read-only when you just want to investigate.
04
Iterate, don't restart.
When the result is close but not right, course-correct in the same conversation â the context you've built is your biggest asset.

Daily Workflow
Why these four steps

Three reasons it always pays off.

01
Explore makes the plan honest.
Letting Claude read the actual code first â not just the description â kills 90% of "fixed something that didn't exist" moments.
02
Plan turns big asks into reviewable work.
A plan is the cheapest checkpoint you'll ever get. Approve the steps, not just the result â you'll catch wrong directions in seconds, not hours.
03
Commit closes the loop.
Small, atomic commits make every change reviewable, revertable, and explainable â even (especially) the ones the agent wrote.

Daily Workflow
Context Management

Three commands. One habit.

Context is the agent's memory of this session. Manage it deliberately and the loop stays sharp; ignore it and Claude starts forgetting â or burning tokens â on irrelevant noise.

/compact
Summarize and continue.
  • Keep the thread alive
  • Trim the long backstory
  • Hold on to the decisions, drop the trial-and-error
/clear
Wipe and restart.
  • Switching to a new task
  • Old context is misleading the agent
  • You want a clean baseline
/context
See what it sees.
  • Inspect the active context window
  • Spot what's eating tokens
  • Know when it's time to /compact
It looks linear

But it's a loop.

Explore â Plan â Code â Commit looks like a checklist. In practice it's a spiral: every code step turns up something the plan missed, every commit raises a new question, every exploration sharpens the next prompt. That's the work.

Start anywhere.
If you have a vague problem, explore. If you have a half-formed solution, plan. The right entry point is whatever unsticks you.
Let code rewrite the plan.
When the implementation surfaces a constraint, update the plan â don't fight it.
Commit early, commit often.
Tiny commits are your undo button. They also make the next prompt easier â the agent can read them.

Module 04 Â· Customization

Five ways to make it yours.

Out of the box, Claude Code is generic. These five primitives are how you teach it your project, your stack, and your standards â so every session starts already aligned.

M
CLAUDE.md
A persistent project memo. Conventions, do-nots, where to look first.
S
Subagents
Delegate specialized jobs â reviews, search, planning â to focused agents.
K
Skills
Reusable, packaged know-how Claude can invoke when the situation matches.
P
MCP
Connect external tools â GitHub, Linear, your DB â over a single open protocol.
H
Hooks
Run shell commands on events â auto-format on save, ping you when a long task ends.
