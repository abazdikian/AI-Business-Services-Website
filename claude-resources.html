Getting Started

Your Claude Code Starter Guide

No coding experience needed. Tap each step.

📋

Step 01

What is Claude Code?

It's Claude — but on your computer

  1. 1Instead of chatting in a browser, Claude works directly on your machine
  2. 2It creates real files, builds apps, and runs actual tasks
  3. 3You don't write code — you describe what you want in plain English
📋

Step 02

Always Start with /plan

Blueprints before construction

  1. 1Type /plan before asking Claude to build anything
  2. 2It maps out the approach, steps, and files it'll create
  3. 3You review and approve before it starts — no surprises
~/ my-project $ claude
> /plan build me a weekly report generator
✓ Planning mode activated...
Here's my approach:
  1. Create project structure
  2. Build data input template
  3. Generate formatted report output
  4. Add export to PDF
Shall I proceed? (y/n)
📂

Step 03

Set Up Your Folder

Future you will thank you

  1. 1Create a project folder with a clear name
  2. 2Open your terminal and navigate to that folder
  3. 3Everything Claude builds stays organised in one place
~ $ mkdir weekly-report
~ $ cd weekly-report
~/weekly-report $ claude
✓ Ready. Working in ~/weekly-report
📁 Everything Claude builds stays right here ✓
🎯

Step 04

Start With Something Annoying

Don't build a SaaS. Fix one thing.

  1. 1Pick a task that's repetitive, boring, or takes too long
  2. 2A weekly spreadsheet. A manual report. A content template.
  3. 3Tell Claude about it. Let it build you a solution.
🔧

Step 05

You'll Break Things

That's the process, not the problem

  1. 1Everyone breaks things. Every day. Including me.
  2. 2Claude doesn't judge — it helps you fix it
  3. 3The only difference is willingness to try again
